# Notas de Pesquisa

Resumo pratico da pesquisa externa para orientar a LP.

## Agent workflows

Fonte: Anthropic, "Building effective agents"

Principios aproveitados:

- Comecar simples e aumentar complexidade apenas quando necessario.
- Usar orquestrador-workers quando o trabalho tem subtarefas imprevisiveis e exige sintese.
- Usar evaluator-optimizer quando ha criterios claros e iteracao melhora o resultado.
- Dar ground truth para agentes: arquivos, prints, builds, logs e contexto real.

Aplicacao na MASI LP:

- Orquestrador central define estrategia e delega.
- Agentes especialistas trabalham em paralelo.
- QA de conversao e QA tecnico fazem ciclo de avaliacao.

URL: https://www.anthropic.com/engineering/building-effective-agents

## Landing page/CRO

Fonte: Unbounce landing page best practices.

Principios aproveitados:

- Manter um objetivo principal de conversao.
- CTA forte e visivel.
- Mostrar produto/servico em acao.
- Usar prova social/prova concreta.
- Remover distracoes.
- Testar e atualizar.

URL: https://unbounce.com/landing-page-articles/landing-page-best-practices/

Fonte: Unbounce, elementos de uma landing page.

Principios aproveitados:

- Proposta unica de valor.
- Hero claro.
- Beneficios.
- Prova.
- CTA.

URL: https://unbounce.com/landing-pages/elements-of-a-winning-landing-page/

## UX e confianca

Fonte: Baymard Institute.

Principios aproveitados:

- UX deve ser orientada por evidencia, nao opiniao.
- Percepcao de seguranca e confianca e influenciada pela apresentacao visual.
- Para uma plataforma empresarial com IA, visual limpo e confiavel importa tanto quanto a promessa.

URLs:

- https://baymard.com/
- https://baymard.com/blog/perceived-security-of-payment-form

## Decisoes para este projeto

- Usar screenshots reais como prova.
- Hero precisa explicar a categoria em uma frase.
- CTA unico: solicitar acesso/aplicar.
- Copy deve responder objecoes antes de pedir conversao.
- Visual deve seguir produto real: claro, verde, cards, rede de inteligencia, agentes.
